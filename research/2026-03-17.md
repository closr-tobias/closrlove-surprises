### 1. Snow Globe (Shake to Reveal)

**Concept:** The screen shows a cozy snow globe scene — a tiny house, trees, warm light inside. The globe appears still. When the user shakes their phone, snow swirls up, and as it settles, love messages appear written in the snowfall pattern. Each shake reveals a new message. After 3-4 shakes, the final message glows inside the house window.

**Why it works for Closr:** Shake is the most intuitive physical gesture — everyone knows how a snow globe works. The physicality bridges digital and tangible. The progression (shake → snow → message settles) creates anticipation. The final house-window message is the emotional payoff: "Come home to me" or "You're my warmth."

**Technical approach:**
- `DeviceMotionEvent` → monitor `accelerationIncludingGravity`, detect shake via threshold (~15 m/s² delta across 500ms)
- iOS 13+ requires `DeviceMotionEvent.requestPermission()` — prompt on first tap
- Snow particles: Canvas 2D, 200-400 particles, each with x/y/vx/vy/radius/opacity
- Shake triggers: randomize all velocities (burst), then apply gravity + slight wind to settle
- Message reveal: particles accumulate along text paths (same pixel-sampling technique from Mar 12 firefly research)
- Globe: CSS `border-radius: 50%` container with `overflow: hidden`, subtle glass reflection via radial gradient overlay
- Fallback (no motion sensor): tap the globe instead, with a wobble animation

### 2. Tilt-to-Pour (Love Potion / Message in a Bottle)

**Concept:** Screen shows a beautifully rendered bottle (potion bottle or glass bottle with a letter inside). The user tilts their phone, and liquid flows with gravity — glowing pink/gold liquid slides to the tilted side. When tilted past ~45°, the liquid "pours out" and forms words on a surface below. Or: tilting a bottle on its side lets a rolled-up message slide out.

**Why it works for Closr:** The gyroscope creates a "magic object" feeling — the liquid genuinely seems to respond to gravity. It's surprising and delightful. The pouring metaphor ("pouring my heart out") is romantically loaded. Message-in-a-bottle evokes distance and longing — perfect for LDR couples.

**Technical approach:**
- `DeviceOrientationEvent` → `beta` (front-back tilt) and `gamma` (left-right tilt)
- Liquid simulation: Simplified — a filled arc/ellipse whose position shifts based on tilt angle
- Advanced: 2D metaball simulation (5-8 blobs that merge, positions driven by tilt-as-gravity)
- Pour threshold: when `beta > 45°` or `gamma > 45°`, trigger pour animation
- Poured liquid: CSS clip-path reveal on the message, expanding from pour point
- Bottle: SVG with glass refraction effect (subtle inner highlight, slight green/amber tint)
- Fallback: drag-to-tilt with touch, or tap-to-pour

### 3. Horizon Panorama (Tilt to Explore)

**Concept:** A wide panoramic scene (sunset, starfield, city skyline at night) wider than the screen. The user tilts their phone left/right to pan across — as if looking through a window. Hidden in the panorama are messages, glowing markers, or constellations that spell words. Feels like exploring a shared view.

**Why it works for Closr:** LDR couples can't share a sunset. This simulates it — "tilt your phone and look at what I see." The exploration rewards curiosity. Finding hidden messages in a landscape feels like a treasure hunt.

**Technical approach:**
- `DeviceOrientationEvent.gamma` → horizontal pan position
- Background: `transform: translateX()` driven by gamma
- Parallax layers (reuse Mar 13 research): 3 depth layers panning at different rates
- Hidden elements: `opacity: 0` → `1` when scrolled into view + paused on for 1s
- Scene width: ~3x viewport width
- Smooth interpolation: `lerp(currentPos, targetPos, 0.08)` for fluid panning
- Fallback: swipe/drag to pan, with momentum

### 4. Compass of Us (Orientation as Navigation)

**Concept:** A beautifully styled compass where N/S/E/W are replaced with relationship milestones ("First Kiss," "First Trip," "Our Future," "Right Now"). The needle tracks real orientation via magnetometer. Physically turn to face a direction → that milestone's story expands into a full scene. Four mini-stories in four directions.

**Why it works for Closr:** "No matter which way I turn, everything points to you." Physical rotation creates a spatial memory palace. Four directions = four scenes = a complete surprise in one block.

**Technical approach:**
- `DeviceOrientationEvent.alpha` → compass heading (0-360°)
- `webkitCompassHeading` on iOS (more reliable)
- Compass rose: SVG with custom labels, needle via `transform: rotate(-alpha deg)`
- Direction detection: 4 quadrants, 1.5s dwell time to trigger content
- Fallback: tap compass directions manually

### 5. Gravity Hearts (Tilt to Play)

**Concept:** Heart particles scattered on screen respond to phone tilt — rolling like marbles. Tilt to guide hearts into a collection zone (jar, chest, heart outline). Each collected heart reveals a word. Collecting all completes the message.

**Why it works for Closr:** Gamification + physics = satisfying. "Collecting love" as metaphor. Impossible to fail, just fun. The completed message is the reward.

**Technical approach:**
- Beta/gamma as gravity vector
- Verlet integration: `ax = gamma * 0.5`, `ay = beta * 0.5`
- Collision: bounce off edges (restitution: 0.6), simplified circle-circle
- Collection zone: circle hitbox, hearts slow and stick on entry
- 8-12 hearts = 8-12 word message
- Canvas 2D for physics, CSS overlay for word reveals
- Fallback: touch-drag to push hearts

### 6. Magic Window (Tilt to Peek)

**Concept:** Screen appears opaque (marble, wood, frosted glass). Tilting the phone moves a "keyhole" — revealing a hidden scene behind: a photo, message, or illustration. The limited viewport forces gradual exploration.

**Why it works for Closr:** Mystery and curiosity. Tilting to peek feels playful. The constrained viewport builds anticipation. Perfect for photo reveals.

**Technical approach:**
- Beta/gamma → circle position
- CSS `clip-path: circle(80px at ${x}px ${y}px)` on hidden content
- Foreground: textured surface with shadow around clip edge
- Parallax on hidden content (slight offset opposite tilt) for depth
- Optional: visited areas remain partially visible
- Fallback: finger position as window — touch-drag to peek

### 7. Candle Blow-Out (Microphone + Tilt)

**Concept:** A candle flame flickers, leaning with phone tilt. If mic is permitted, blowing dims and extinguishes it. Darkness reveals glow-in-the-dark text. "Make a wish for us."

**Why it works for Closr:** Birthday candles are universal ritual. Blowing on your phone is silly and magical. Dark-to-glow reveal is dramatic. Perfect for birthday/anniversary surprises.

**Technical approach:**
- Mic: `getUserMedia({ audio: true })` → `AnalyserNode` → amplitude monitoring
- Sustained amplitude >1.5s = "blow" detected
- Flame: CSS gradient + `filter: blur()`, animated by tilt + audio
- Smoke after blow-out: Canvas particles, rising with drift
- Glow text: multi-layer `text-shadow`, fade in after 1s darkness
- Fallback: long-press flame (no mic needed)

## Device Sensor API Reference for Closr

### Permission Handling (Critical for iOS)
```javascript
async function requestMotionPermission() {
  if (typeof DeviceMotionEvent.requestPermission === 'function') {
    const permission = await DeviceMotionEvent.requestPermission();
    return permission === 'granted';
  }
  return true; // Android grants automatically
}
```
- Always gate behind a user tap (iOS requirement)
- Design "tap to begin" as part of the scene, not a system dialog

### Orientation Values
- `alpha` (0-360): compass heading (Z axis rotation)
- `beta` (-180 to 180): front-back tilt
- `gamma` (-90 to 90): left-right tilt

### Smoothing (Essential)
Raw sensor data is jittery. Always smooth:
```javascript
let smoothGamma = 0;
window.addEventListener('deviceorientation', (e) => {
  smoothGamma = smoothGamma * 0.85 + e.gamma * 0.15;
});
```

## Fallback Strategy

Every sensor-driven block MUST have a touch/tap fallback:

- **Shake** → Tap rapidly or tap a "shake" button
- **Tilt (gravity)** → Touch-drag to set gravity direction
- **Tilt (pan)** → Swipe/drag to pan
- **Orientation (compass)** → Tap compass directions
- **Microphone (blow)** → Long-press
