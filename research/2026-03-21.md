## Why 3D/Depth for Closr

Flat screens create emotional distance. When a scene has *depth* — layers that move at different speeds, objects that rotate in 3D space, reveals that feel like opening a physical object — it triggers a sense of intimacy and physicality. For a couples app where you're sending love across a screen, making that screen feel like a *window into a space* rather than a flat page is powerful.

## Idea 1: Love Letter Envelope — 3D Open & Unfold

**Concept:** A sealed envelope rendered in CSS 3D. The flap lifts open with perspective transform, a letter slides out, then unfolds in stages — each fold revealing a line of text.

**Why it works for Closr:** The physicality of opening a real letter. Every couple remembers the first note, the first card. This makes digital feel analog. The anticipation of each unfold builds emotional tension.

**Technical approach:**
- CSS `perspective` on container (800-1200px)
- Envelope flap: `transform: rotateX(-180deg)` with `transform-origin: top`
- Letter slide-out: `translateY` + slight `rotateX` for paper curl effect
- Multi-stage unfold: chain CSS animations with `animation-delay`
- Paper texture via subtle `background-image` or CSS noise gradient
- Handwritten-style font for the message content

**Differentiation from `paper-unfold.html`:** Paper-unfold is a flat 2D unfolding. This is a full 3D envelope with flap physics, letter extraction, and multi-fold reveal — a complete physical object simulation.

## Idea 2: Memory Diorama — Parallax Depth Box

**Concept:** A scene built like a shadow box / diorama with 3-5 layers at different Z-depths. Foreground elements (flowers, frames), midground (photo of the couple), background (sky, pattern). Tilting the phone (gyroscope) or dragging shifts the perspective, revealing depth between layers.

**Why it works for Closr:** It turns a flat photo into a *place*. The couple's photo lives inside a little world. It feels precious, like a keepsake you'd put on a shelf.

**Technical approach:**
- Container with `perspective: 1000px` and `transform-style: preserve-3d`
- Each layer: absolutely-positioned div with different `translateZ` values (-100px, 0, 100px, 200px)
- Gyroscope input (DeviceOrientationEvent) maps to container `rotateX`/`rotateY` (clamped ±15°)
- Fallback: touch drag maps to rotation
- Subtle shadow on each layer that shifts with perspective
- Themeable layers (cherry blossoms, snow, autumn leaves)

**Reference:** Apple's parallax wallpapers; shadow box craft tradition; Instagram's old 3D photo feature.

## Idea 3: Rotating Photo Cube

**Concept:** A CSS 3D cube where each face shows a different photo or message. Auto-rotates slowly, swipe/drag spins it manually. Each face is a "chapter" — first date, first kiss, first trip, etc.

**Why it works for Closr:** Six faces = six memories in one compact, tactile object. The spinning creates discovery. Works as a standalone Tier 1 surprise — the cube IS the surprise.

**Technical approach:**
- 6 divs with `transform-style: preserve-3d`
- Each face: `rotateY(0/90/180/270deg) translateZ(half-width)` + `rotateX(±90deg)` for top/bottom
- Auto-rotation via `@keyframes`; drag-to-spin with momentum (rAF + decay)
- Photos as `background-image` with cover
- Optional: tap a face to zoom into that memory

## Idea 4: Pop-Up Card — Paper Craft in CSS

**Concept:** A greeting card that opens in 3D — front cover swings open, paper-craft elements pop up from the fold (hearts, couple silhouette, text). Pure CSS 3D with transform-origin along the card's spine.

**Why it works for Closr:** Greeting cards are THE romantic gesture. The pop-up mechanic creates surprise — you don't know what's inside until it opens.

**Technical approach:**
- Card halves joined at center with `transform-origin: left center`
- Front half: `rotateY(-160deg)` to swing open
- Pop-up elements at fold line: initially `rotateX(90deg)`, spring up as card opens
- Elements: SVG silhouettes (couple, hearts, cityscape)
- Ribbon/bow exterior as "pull to open" cue
- Sound: paper unfolding, gentle "pop"

## Idea 5: Portal / Window Effect — Looking Through

**Concept:** The screen becomes a window/portal. A frame (window pane, keyhole, telescope) sits in foreground; behind it, a scene with depth — clouds, stars, landscape. Parallax between frame and scene creates the illusion of looking *through* the phone.

**Why it works for Closr:** "I wish I could look through my screen and see you." This literalizes that feeling. The keyhole adds mystery/intimacy. The telescope works for LDR couples — "looking across the distance."

**Technical approach:**
- Foreground frame with cutout via CSS `clip-path` or `mask-image`
- Background: multiple parallax layers (`translateZ` or gyroscope-linked)
- Shaped windows: keyhole, heart, telescope circle
- Tilt phone to shift what's visible through the window

**Reference:** Apple Vision Pro's window metaphor; View-Master toys.

## Idea 6: Depth-of-Field Text — Focus Pull

**Concept:** Multiple text lines at different "depths." All blurred except one. Tap/scroll to pull focus to the next line — like a camera rack focus. Background blurs as foreground sharpens.

**Why it works for Closr:** Cinematic reading experience. Each line gets its own moment. Blur-to-sharp = "seeing clearly what someone means to you." Beautiful for love letters or vow-style messages.

**Technical approach:**
- Each line with different `translateZ` + `filter: blur()` mapped to distance from focal plane
- Focal plane shifts on tap/scroll via CSS transitions or GSAP
- Slight scale change with depth for realism
- Soft bokeh particle background for full camera effect

---

## Idea 7: Folding Screen / Accordion Reveal

**Concept:** Content behind folded panels — like a paper fan. Tap/swipe to unfold each panel. Each panel at a slightly different angle creates a zigzag 3D effect.

**Why it works for Closr:** Sequential reveal with physicality. Perfect for "reasons I love you" lists — one reason per fold.

**Technical approach:**
- Panels with alternating `transform-origin` (left/right)
- Folded: alternating `rotateY(±80deg)` zigzag
- Unfold sequentially to `rotateY(0)`
- `backface-visibility: hidden`; shadows between panels

## Cross-Cutting Technical Notes

### Performance on Mobile
- CSS 3D transforms are GPU-accelerated — generally performant
- `will-change: transform` on animated elements
- Limit ~10 3D-transformed elements per scene
- `transform-style: preserve-3d` nesting can be finicky — keep hierarchy shallow

### Key CSS Properties
- `perspective`: 800-1200px sweet spot
- `transform-style: preserve-3d`: children in parent's 3D space
- `backface-visibility: hidden`: essential for flips and folds
- `transform-origin`: controls rotation pivot
