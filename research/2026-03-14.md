## Focus: Sound Design & Audio-Reactive Visuals for Emotional Scenes

Previous research covered: Typography animations (Mar 8), Particle systems (Mar 12), Scroll-driven storytelling (Mar 13). This session explores **sound as a first-class design element** — how audio transforms a visual scene into a multisensory experience, and how visuals can react to sound in real-time.

Existing blocks reviewed: 23 blocks. None currently use audio or sound-reactive visuals. This is an untapped dimension for Closr surprises.

### 1. Audio-Reactive Heartbeat Waveform

**Concept:** A central waveform visualization that pulses to a real heartbeat sound (60-80 BPM). The waveform morphs between a flat line and a heart shape at each beat. Text messages appear synced to the beat peaks.

**Why it works for Closr:** The heartbeat is the most universal symbol of love. Syncing visuals to actual audio creates a visceral, almost physical connection. In an LDR context, it simulates the intimacy of hearing your partner's heartbeat.

**Technical approach:**
- Web Audio API: Load heartbeat `.mp3`, use `AnalyserNode.getByteTimeDomainData()` for waveform
- Canvas 2D: Draw waveform as bezier curves, morph control points toward heart-shape on beat peaks
- Beat detection: Simple threshold on amplitude data
- Haptic: `navigator.vibrate([100])` on each beat for mobile
- Fallback: CSS animation-only mode if audio blocked by autoplay policy
- Audio source: Short royalty-free heartbeat loop (~10s), base64 data URI

### 2. Vinyl Record / Music Box Scene

**Concept:** A spinning vinyl record plays a short melody. As the music plays, love messages appear as liner notes. Record rotation synced to audio playback. User taps to "lift the needle" and pause.

**Why it works for Closr:** "Our song" is a universal couple concept. The vinyl aesthetic adds nostalgia. The physical interaction (lift/drop needle) makes it feel like a real object.

**Technical approach:**
- CSS `transform: rotate()` driven by `AudioContext.currentTime`
- SVG vinyl with grooves (concentric circles, slight wobble)
- Tone arm as separate SVG, `pointer-events: all` for pause interaction
- Melody: Tone.js synthesized music-box melody (~2KB) or base64 piano loop
- Vinyl crackle: White noise via `createBufferSource()`, low-pass filtered, low gain

### 3. Whispering Text (ASMR Reveal)

**Concept:** Dark screen. A soft whisper plays a phrase. Words materialize on screen as glowing particles that coalesce into letters — as if sound creates the text. Each phrase arrives with breathy, intimate audio.

**Why it works for Closr:** ASMR-style audio creates intense intimacy. Darkness + whisper feels like your partner is right next to you. Quiet and close, the opposite of flashy.

**Technical approach:**
- Pre-recorded whisper clips (3-5 phrases, ~2-3 seconds each)
- `PannerNode` for stereo positioning (voice moves subtly left-right)
- Canvas particles (`globalCompositeOperation: 'lighter'`) drifting toward letter positions
- Letter positions from off-screen canvas `measureText()` + pixel sampling
- Glow: Multiple layered `text-shadow` values
- 🎧 prompt before starting — designed for headphones

### 4. Rain Melody (Tap-to-Create-Music)

**Concept:** Raindrops fall on screen, each playing a soft piano note (pentatonic scale — can't sound bad). User taps to create their own raindrops + notes. A message is revealed in the puddles/splashes.

**Why it works for Closr:** The user becomes a co-creator. Pentatonic constraint means any combination sounds pleasant — no wrong way to play. Message-in-puddles payoff rewards patience.

**Technical approach:**
- Tone.js `Synth` with pentatonic scale (C4, D4, E4, G4, A4)
- Raindrop: CSS animation (fall from random X, accelerate via cubic-bezier)
- Note selection: Y-position → pitch, X-position → stereo pan
- Puddle text: `clip-path` reveal driven by accumulated splash count
- Reverb: Tone.js `Reverb` with ~3s decay for dreamy atmosphere
- Limit: 10 simultaneous drops + 5 user-tapped drops

### 5. Audio Message Waveform Envelope

**Concept:** Sender records a voice message (or AI generates TTS). Displayed as a beautiful waveform — not WhatsApp's ugly gray bar, but a colorful, flowing shape inside a love-letter envelope. Waveform lights up progressively during playback.

**Why it works for Closr:** Hearing your partner's actual voice, beautifully presented, elevates a voice note from utility to art. The envelope metaphor ties audio to the love letter tradition.

**Technical approach:**
- `MediaRecorder` API for recording (sender side)
- `AnalyserNode.getByteFrequencyData()` for waveform visualization
- Canvas: Gradient-filled waveform bars, progressive highlight on playback
- Envelope SVG: Flap opens via CSS `transform: rotateX()` with perspective
- Opus codec @ 24kbps → ~3KB/second, 15s clip under 50KB

### 6. Sound Wave Portrait

**Concept:** Classic "sound wave art" (waveform of "I love you") rendered as interactive art — styled as a mountain range, city skyline, or abstract piece. Tap to play the actual audio behind the art.

**Why it works for Closr:** Sound wave prints are already a popular romantic gift (Etsy shops sell them). Making it interactive — where the art *plays* — adds the dimension those prints lack. A living keepsake.

**Technical approach:**
- Pre-analyze audio → extract amplitude envelope (downsample to ~200 points)
- Render as SVG `<path>` with smooth bezier interpolation
- Style variants: mountain (filled, gradient sky), skyline (rectangular bars), organic (mirrored wave)
- Tap-to-play with playback position highlight
- CSS `@media print` stylesheet for physical keepsake option

### 7. Binaural Beats Meditation Scene

**Concept:** Abstract morphing gradient blobs or aurora borealis paired with binaural beats (alpha waves, 8-12 Hz). Soft affirmations and memories appear. A 60-90 second shared meditation moment.

**Why it works for Closr:** LDR couples lack shared calm moments. This creates one. The wellness dimension is trending. No other couples app offers meditative surprises.

**Technical approach:**
- Two `OscillatorNode`s: left ear 200Hz, right ear 210Hz (10Hz alpha difference)
- Soft pad: Low-pass filtered noise + gentle sine sweep
- Visuals: CSS `filter: blur()` on overlapping divs with morphing border-radius
- Text: Very slow fade (3s transitions), large serif font
- End state: "Take a deep breath together 💙" with partner's name

## Audio Implementation Notes for Closr Blocks

### Autoplay Policy Handling
Every audio block needs:
1. A "tap to begin" prompt (designed into the scene, not an ugly button)
2. `AudioContext.resume()` inside a user gesture handler
3. Visual-only fallback that still works without sound
4. 🔊 icon indicating "best with sound"

### File Size Budget
- Voice clips: Opus @ 24kbps → ~3KB/second
- Short loops: Base64-encode into HTML (under 50KB)
- Synthesized audio (Tone.js / Web Audio oscillators): Zero file size
- **Total audio budget per block: under 100KB**

### Haptic Pairing
`navigator.vibrate()` on Android Chrome pairs subtle vibrations with audio beats. iOS Safari doesn't support it — degrade gracefully.
