## Why Tactile/Physical for Closr?

Romance lives in the handmade. A handwritten letter beats an email. A pressed flower beats a stock photo. Digital scenes that simulate **physical craft materials** trigger nostalgia and intimacy — they feel like someone *made* something rather than clicked a button. This is the opposite of the slick, sterile UI trend. It says: "I cared enough to make this feel real."

Existing blocks like `paper-unfold`, `popup-card`, and `love-letter-typewriter` already touch this space, but there's a whole universe of physical material metaphors unexplored.

## Idea 1: Wax Seal Reveal

**Concept:** A wax seal sits on textured parchment. User taps/presses the seal — it cracks and crumbles apart with satisfying physics, revealing a message or image beneath. The seal can be personalized (initials, a heart, a custom symbol).

**Why it works for Closr:**
- Wax seals are universally associated with love letters and secrecy
- The "breaking" interaction is tactile and satisfying (like popping bubble wrap)
- Creates anticipation — what's under the seal?
- The seal itself can be personalized with the couple's initials

**Technical approach:**
- SVG seal with fracture lines pre-computed (Voronoi tessellation)
- On tap, fragments animate outward with slight rotation and gravity using CSS transforms + JS physics
- Parchment texture via CSS `background-image` with noise grain overlay
- Seal texture: radial gradient + `box-shadow` for depth, `filter: drop-shadow` for wax sheen
- Fragment physics: simple spring-damper system, each piece gets random velocity + angular momentum
- Sound: a satisfying crack/snap SFX on break

**Not covered by existing blocks:** `paper-unfold` unfolds paper but doesn't simulate material destruction. `scratch-reveal` is closest but uses a different metaphor (lottery card vs. wax seal).

## Idea 2: Embossed / Letterpress Text

**Concept:** Text appears as if physically stamped or embossed into thick cotton paper. As the user tilts their phone (gyroscope) or scrolls, the light source shifts, revealing the depth of the embossing through shadow changes. Text could be a love quote, pet name, or date.

**Why it works for Closr:**
- Embossing feels luxurious, like a wedding invitation or premium stationery
- The gyroscope interaction makes it feel *real* — you're holding a physical object
- Subtle but stunning; the kind of detail that makes someone go "wait, how..."
- Perfect for short, impactful text: names, dates, vows, one-liners

**Technical approach:**
- CSS `text-shadow` with multiple layers (light + dark) to simulate raised/pressed effect
- Gyroscope via `DeviceOrientationEvent` to shift shadow offsets in real-time
- Paper texture: high-res noise pattern, slight warm tint (#f5f0e8), `background-blend-mode: multiply`
- For pressed (debossed) look: invert shadow direction, add subtle `inner-shadow` via pseudo-elements
- Fallback for non-gyro devices: shadow shifts on scroll position or gentle CSS animation loop
- Optional: add a subtle paper fiber pattern using SVG `feTurbulence` filter

**Not covered by existing blocks:** `love-letter-typewriter` types text but on flat screen. This is about *dimensionality* — text that lives in the material.

## Idea 3: Torn Paper Collage

**Concept:** The scene builds up as torn pieces of paper drift in and land, assembling a photo or message like a handmade scrapbook page. Each piece has realistic torn edges (rough, fibrous). Pieces may have different textures — notebook paper, kraft paper, watercolor paper, old map fragments.

**Why it works for Closr:**
- Scrapbooking is deeply associated with preserving memories and love
- The assembly animation creates wonder — "what is this becoming?"
- Each piece arriving feels like a thought or memory coming together
- Mixed textures add visual richness and a handcrafted aesthetic

**Technical approach:**
- Torn edges: SVG `clipPath` with irregular, organic paths (can be procedurally generated with Perlin noise along edges)
- Each piece: `div` with paper texture background, clipped to torn shape, positioned absolutely
- Animation: pieces enter with slight float + rotation, land with a soft "thud" (transform + subtle scale bounce)
- Staggered timing via CSS `animation-delay` or GSAP timeline
- Different paper types: vary `background-color`, grain density, opacity
- Final state: pieces form a recognizable shape (heart, photo frame, text)
- Optional: tape strips or pin decorations at corners using CSS pseudo-elements with slight rotation

**Not covered by existing blocks:** `love-jigsaw` does puzzle assembly but with clean geometric pieces. This is organic, messy, *human*.

## Idea 4: Fabric Fold / Silk Curtain Reveal

**Concept:** A silky fabric curtain covers the scene. The user swipes or drags to pull it aside, revealing content beneath. The fabric moves with realistic draping physics — it bunches, wrinkles, catches light. Think unveiling a painting or pulling back a theater curtain.

**Why it works for Closr:**
- Theater curtain = dramatic reveal, perfect for surprise moments
- Silk/satin has inherently romantic, intimate associations
- The drag interaction makes the user *physically reveal* their surprise
- Rich visual — fabric simulation looks premium and impressive

**Technical approach:**
- WebGL mesh with cloth physics (verlet integration, simple spring model)
- Fabric rendered as a grid of points with texture mapping
- User drag applies force to nearest fabric points
- Light simulation: normal-based shading for silk sheen (Phong model)
- Simpler fallback: CSS `clip-path` animation that reveals content from the drag point, with a fabric texture overlay that slides away with CSS transform
- Gradient overlay on fabric for depth (lighter where "raised," darker in folds)
- Performance: keep grid resolution low (20x30 points), use `requestAnimationFrame`

**Not covered by existing blocks:** Nothing in the current set does fabric/cloth simulation. `paper-unfold` is rigid folding; this is soft, flowing material.

## Idea 5: Pressed Flower / Botanical Press

**Concept:** Dried, pressed flowers fade in one by one on aged paper, as if someone opened a botanical press or an old book with preserved flowers. Each flower is slightly translucent, flattened, with visible veins. A handwritten label appears beside each: "The first flower you gave me," "From our park," etc.

**Why it works for Closr:**
- Pressed flowers are one of the oldest romantic gestures in existence
- Combines nature, nostalgia, and personal meaning
- Each flower can represent a memory or milestone
- The aged paper + botanical aesthetic is timeless and gender-neutral

**Technical approach:**
- Flowers as semi-transparent PNGs or SVGs with reduced saturation and warm color shift
- Paper background: parchment texture with foxing spots (age marks) via CSS radial gradients
- Each flower fades in with slight scale (1.02 -> 1.0) and opacity (0 -> 0.85, not full — keep translucent)
- Handwritten labels: web font like "Caveat" or "Dancing Script," animated with `stroke-dashoffset` on SVG text paths
- Gentle parallax: flowers shift slightly on scroll/tilt, creating depth
- Optional: a book spine on the left edge, suggesting this is a page in a journal

**Not covered by existing blocks:** `bloom-garden` grows living flowers; this is about *preserved* flowers — a completely different emotional register (nostalgia vs. growth).

## Idea 6: Stitch & Thread Animation

**Concept:** A message or design is "stitched" onto fabric in real-time — a needle and thread traces cross-stitch or embroidery patterns, spelling out words or drawing a heart. The fabric texture is visible beneath, slightly puckering around each stitch.

**Why it works for Closr:**
- Embroidery = patience, care, handcraft — "I made this stitch by stitch for you"
- The animation is mesmerizing — watching the pattern form is deeply satisfying
- Cross-stitch has a retro/cozy/cottage-core aesthetic that's hugely popular
- Can encode personal messages: names, dates, "I love you" in stitches

**Technical approach:**
- Canvas-based: draw thread as bezier curves with slight sag between stitch points
- Grid system for cross-stitch pattern (each X is two overlapping diagonal lines)
- Animation: draw each stitch sequentially with `requestAnimationFrame`, ~100ms per stitch
- Thread texture: slight color variation along length, subtle shadow beneath thread
- Fabric texture: CSS grid of tiny dots or crosshatch pattern with warm linen color
- Needle: small SVG element that leads the thread, rotates to match stitch direction
- Puckering effect: subtle `filter: displacement-map` around recent stitches

**Not covered by existing blocks:** Nothing in the current set does textile/needlework simulation. This is an entirely new material metaphor.

## Idea 7: Candle Wax Drip Message

**Concept:** A lit candle drips wax that pools and forms letters/words on a dark surface. The wax is warm-colored, semi-translucent, with realistic pooling physics. The candlelight flickers and casts dancing shadows. Intimate, almost ceremonial.

**Why it works for Closr:**
- Candlelight is the universal symbol of romance and intimacy
- Watching wax form words is hypnotic and unique — no one's seen this in an app
- The flickering light creates ambient mood even before the message forms
- Works especially well for evening/night surprises

**Technical approach:**
- Candle: CSS gradient + `filter: blur` for soft glow, flame via animated SVG or CSS keyframes with randomized flicker
- Wax drips: Canvas fluid simulation (simplified — just gravity + surface tension for pooling)
- Letter formation: pre-defined paths that wax "flows" along, using fluid particle system
- Wax appearance: warm translucent gradient (#c4451d to #e8a54b), `mix-blend-mode: screen` for glow
- Shadows: dynamic `box-shadow` or `filter: drop-shadow` offset that oscillates with flame flicker
- Dark surface: near-black with subtle wood grain texture
- Sound: gentle crackling/ambient fire loop

**Not covered by existing blocks:** `whisper-glow` has ambient light effects but no candle/wax simulation. This is specifically about *molten material forming meaning*.

## Cross-Cutting Technical Notes

### Texture Generation Without Assets
Many of these ideas need textures (paper grain, fabric weave, wood grain). Rather than loading image assets (which bloat file size), generate procedurally:
- **Paper grain:** SVG `feTurbulence` + `feColorMatrix` — creates convincing noise
- **Fabric weave:** CSS repeating `linear-gradient` at 0deg and 90deg with thin lines
- **Wood grain:** SVG `feTurbulence` with high `baseFrequency` x and low y
- **Parchment foxing:** Multiple CSS `radial-gradient` with random positions and warm brown colors

### Haptic Feedback
All tactile ideas benefit enormously from `navigator.vibrate()` on supported devices:
- Wax seal crack: short strong pulse `vibrate(50)`
- Paper tear: series of tiny pulses `vibrate([10, 5, 10, 5, 10])`
- Stitch landing: gentle tap `vibrate(15)`
- This makes the "physical" illusion dramatically more convincing

### Performance Considerations
- Cloth simulation and fluid dynamics are GPU-intensive — always provide CSS-only fallbacks
- Texture generation via SVG filters should be cached (render once to canvas, use as `background-image`)
- Keep particle counts low for mobile: 50-100 max for wax/paper fragments
